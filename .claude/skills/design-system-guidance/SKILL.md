# Design System Guidance

When triggered:
1. Read `docs/design-system.md` first.
2. Treat tokens and component rules as constraints when proposing UI changes.
3. Ensure accessibility and performance fallbacks are preserved (especially for blur/backdrop effects).

If the project has no `portal/`, apply this skill only to docs or future portal scaffolding.
