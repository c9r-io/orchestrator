# Orchestrator Phase 3 Task 04

## Title

为历史事件与旧任务提供更系统的观测兼容与回填策略

## Goal

减少旧任务在新控制面/trace 中只能显示 `unknown` 或降级信息的情况，为历史事件建立更清晰的兼容与回填策略。

## Problem

现在新的观测语义已经更准确，但历史任务仍有兼容层：

- 缺少 `step_scope` 的旧事件只能显示 `scope=unknown`
- 一些旧 trace 字段只能降级推断或保守展示
- 长期来看，系统里会同时存在“新语义任务”和“旧语义任务”

如果长期不治理：

- 观测输出会持续混有大量 `unknown`
- 新老任务分析体验割裂
- 使用者会逐渐失去对历史数据的信心

## Scope

- 明确哪些历史事件需要兼容回填
- 为可安全回推的旧事件提供受控 backfill 或显示增强策略
- 降低历史任务的观测降级比例

## Out Of Scope

- 不重写整个 events 表
- 不做高风险、不可逆的大规模历史迁移
- 不篡改无法可靠推断的旧数据

## Acceptance Criteria

1. 历史任务中的高频观测降级（如 `scope=unknown`）有明确治理路径。
2. 对可安全回推的字段，系统可提供受控 backfill 或更强兼容显示。
3. 对无法安全回推的旧数据，系统也能更明确地说明“为什么仍未知”。

## Suggested Verification

- 选取一批升级前任务进行 `task trace --verbose`
- 对比治理前后的 `unknown` 降级比例
- 验证新老任务在关键观测字段上的可读性差异是否缩小
