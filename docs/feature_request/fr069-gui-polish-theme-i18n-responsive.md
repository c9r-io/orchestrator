# FR-069: GUI 体验打磨 — 主题切换 / 动画 / i18n / 响应式 / 构建分发

**状态**: Proposed

## 背景

GUI 的核心功能已就绪，但距离产品级体验仍有差距。本 FR 聚合 P2 级体验打磨项，可分批推进。

## 目标

1. 实现 Light/Dark 主题切换与持久化
2. 完善等待动画和过渡效果
3. 预留 i18n 国际化接口
4. 实现响应式布局适配
5. 验证 `cargo tauri build` 产品构建并优化分发
6. 完善 DAG 可视化和日志功能

## 设计方案

### 1. 主题切换

- 顶栏右侧添加主题切换按钮（🌞/🌙 图标）
- 切换时在 `<html>` 上设置 `data-theme="dark"` 属性
- 持久化到 `localStorage("theme")`（与 `docs/design-system.md` 约定一致）
- 启动时读取持久化值，默认跟随系统 `prefers-color-scheme`

### 2. 等待动画

**许愿提交后分阶段提示**：
```
→ "正在理解你的需求..."（0-3s）
→ "正在设计功能方案..."（3-8s）
→ "正在撰写 FR 文档..."（8s+）
```

**页面切换**：200ms ease-out 滑动过渡
**状态变更**：300ms 颜色渐变
**骨架屏**：数据加载中显示灰色占位块

### 3. i18n 预留

- 抽取所有硬编码中文字符串到 `gui/src/lib/i18n.ts` 常量文件
- 默认导出中文，预留结构支持 `react-intl` 或类似库切换
- 不在本 FR 中实现多语言，仅做抽取

### 4. 响应式布局

- 设置最小窗口宽度 960px（`tauri.conf.json` → `minWidth`）
- 宽度 < 1000px 时：许愿列表卡片纵向堆叠、专家面板全宽
- 宽度 ≥ 1200px 时：专家面板可作为右侧边栏展开

### 5. 产品构建与分发

- 验证 `cargo tauri build` 在 macOS 上生成 `.dmg`
- 设计正式应用图标（替换 32×32 占位图标为 1024×1024 RGBA）
- 配置 `tauri.conf.json` bundle 签名选项（macOS code signing 预留）
- 确保产品构建的二进制内嵌前端静态文件、不依赖 Vite

### 6. DAG 可视化增强

当前 ExpertWorkflow 只是顺序列表。增强为：
- 使用 `TaskInfo.graph_debug` 中的 `effective_graph_json` 解析步骤依赖
- SVG 绘制节点 + 箭头，支持并行分支的横向展示
- 点击节点高亮对应 Item 和日志段

### 7. 日志区功能增强

- 关键字搜索（输入框 + 高亮匹配行）
- 日志条数限制：最多保留最近 500 条，更早的可上滚加载
- 暂停自动滚动按钮（用户向上滚动时自动暂停）

## 成功标准

1. Light/Dark 主题切换正常，持久化生效
2. 许愿提交后显示分阶段等待提示
3. 所有中文字符串已抽取到 i18n 常量文件
4. 窗口缩小至 960px 时布局不破碎
5. `cargo tauri build` 生成可安装的 macOS .dmg
6. DAG 可视化展示并行分支（非纯顺序列表）
7. 日志区支持搜索和自动滚动暂停

## 约束

1. 不引入新的 CSS 框架，继续使用 Liquid Glass 设计系统
2. i18n 仅做字符串抽取，不引入第三方库
3. 各子项可独立实现，不阻塞彼此
