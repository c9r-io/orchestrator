# FR-018 用户指南编译验证对齐

- ID: `FR-018`
- Priority: `P1`
- Status: `Proposed`
- Owner: `orchestrator core`

## 背景

项目用户指南（`docs/guide/`）记录了 CLI 命令、子命令、参数、输出格式等信息。随着代码快速迭代，指南中的命令示例、参数名、子命令结构容易与实际实现产生漂移。此前已多次出现指南中记录的命令在实际编译产物中不存在、参数名已更改、或输出格式已变化的情况。

手动逐条核对成本高且易遗漏。本 FR 旨在建立一个**编译驱动的文档治理任务**：实际编译项目，运行 CLI `--help` 输出，逐条对比指南内容，自动发现并修正漂移。

## 问题陈述

- 用户指南中的 CLI 命令示例可能引用不存在的子命令或已废弃的参数。
- `--help` 输出的参数描述、默认值与指南描述不一致。
- 新增的子命令或参数未同步到指南中。
- 缺乏自动化机制来检测文档与代码的偏差。

## 目标

- 建立一个可重复执行的文档对齐流程，以实际编译产物为 ground truth。
- 覆盖所有 `docs/guide/` 中引用的 CLI 命令和参数。
- 输出对齐报告，明确列出差异项和建议修正。
- 对已确认的差异直接修正文档。

## 非目标

- 不在本 FR 中修改 CLI 本身的命令结构或参数命名。
- 不建立 CI 自动门禁（可作为后续增强）。
- 不覆盖非 CLI 相关的文档内容（架构文档、设计文档等）。

## 范围

### In

- `docs/guide/07-cli-reference.md` 中所有命令和参数
- `docs/guide/01-quickstart.md` 至 `06-self-bootstrap.md` 中出现的命令示例
- 中文版本 `docs/guide/zh/` 对应内容
- 实际编译 `cargo build` 后的 CLI 二进制 `--help` 输出
- 子命令树遍历：`orchestrator --help`、`orchestrator <subcmd> --help`、`orchestrator <subcmd> <sub-subcmd> --help`

### Out

- API/gRPC 接口文档对齐（属于独立治理范畴）
- 非 orchestrator CLI 的工具文档

## 需求

### 1. 编译与命令发现

- 执行 `cargo build` 确保最新代码编译通过。
- 遍历 CLI 二进制的子命令树，收集所有命令路径及其 `--help` 输出。
- 解析 `--help` 输出，提取：子命令列表、参数名、参数类型、默认值、是否必填。

### 2. 文档扫描与比对

- 解析 `docs/guide/*.md` 中的代码块和命令引用。
- 提取文档中声明的命令路径、参数名、参数描述。
- 逐项对比，产出差异报告：
  - **Missing in doc**: 代码中存在但文档未记录的命令/参数
  - **Missing in code**: 文档中记录但代码中不存在的命令/参数
  - **Mismatch**: 参数名、类型、默认值或描述不一致

### 3. 自动修正

- 对已确认的差异项，直接修改对应文档文件。
- 修正内容包括：更新参数名、补充缺失命令、删除已废弃命令、修正默认值描述。
- 中英文文档同步修正。

### 4. 对齐报告

- 输出结构化报告（Markdown 格式），包含：
  - 检查时间戳
  - 编译状态
  - 逐命令对比结果
  - 已自动修正项
  - 需人工确认项

## 验收标准

- 流程可通过单条命令或 skill 触发执行。
- 编译失败时提前终止并输出错误信息。
- 对齐报告准确反映文档与实现的所有差异。
- 自动修正后的文档与 `--help` 输出一致。
- 中英文版本保持同步。
- 流程可重复执行，幂等无副作用。

## 风险与缓解

- 风险：`--help` 输出格式变化导致解析失败。
  - 缓解：使用 clap 的结构化输出能力（若可用），或对解析逻辑做容错处理。
- 风险：自动修正破坏文档中的上下文说明或示例叙述。
  - 缓解：仅修正命令/参数的事实性描述，保留周边叙述文本不变；修正后生成 diff 供人工 review。
- 风险：中英文文档结构不完全对称，导致同步修正遗漏。
  - 缓解：以英文版为基准，中文版按段落映射同步。
