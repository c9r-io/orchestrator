# QA-121: GUI 体验打磨 — 视觉与交互

**关联**: FR-069, Design Doc 079
**分类**: GUI / UX
**优先级**: P2

## Scenario 0: 入口可见性

**入口**: 启动 GUI 后，顶部导航栏右侧有主题切换按钮（月亮/太阳图标）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 启动 GUI | 顶栏右侧显示主题切换按钮 |
| 2 | 点击主题按钮 | 弹出 Light/Dark 切换选项或直接切换 |

**预期**: 主题切换入口可见且可操作。

---

## 场景 1: Light/Dark 主题切换

**前置条件**: GUI 已启动

1. 观察顶栏导航右侧有主题切换按钮（月亮或太阳图标）
2. 点击按钮，验证界面在 Light/Dark 模式间切换
3. 刷新页面（或重启应用），验证主题选择被持久化
4. 清除 `localStorage("theme")`，重启应用，验证跟随系统 `prefers-color-scheme`

**预期**: 切换即时生效，持久化正确，默认跟随系统偏好。

## 场景 2: 等待动画与过渡效果

1. 进入许愿池，提交一个许愿
2. 观察分阶段提示："正在理解你的需求..." → "正在设计功能方案..." → "正在撰写 FR 文档..."
3. 观察 spinner 使用 CSS 类动画（非内联 keyframes）
4. 进入进度观察页，loading 状态显示骨架屏（灰色脉冲占位块）
5. 打开删除确认对话框，观察有 fade-in + scale 入场动画

**预期**: 动画流畅、过渡自然、loading 使用骨架屏。

## 场景 3: DAG 可视化

1. 进入一个包含工作流步骤的任务详情
2. 打开专家模式 → 工作流 tab
3. 观察 SVG 绘制的 DAG 图：节点为圆角矩形，箭头连接
4. 如有并行步骤（相同 order_no），验证横向排列

**预期**: DAG 图以 SVG 渲染，支持并行分支横向展示。

## 场景 4: 日志区功能

1. 进入任务详情，点击「追踪」开始接收日志
2. 在搜索框输入关键字，验证匹配行被过滤并高亮
3. 向上滚动日志区域，验证出现「回到底部」按钮
4. 点击「回到底部」，验证自动滚动恢复
5. 当日志超过 500 条时，验证显示截断提示

**预期**: 搜索高亮、自动滚动暂停/恢复、500 条限制均正常工作。

## Checklist

- [ ] S1: Light/Dark 主题切换即时生效，持久化和系统偏好跟随正常
- [ ] S2: 分阶段提示动画正常，spinner 使用 CSS 类，骨架屏显示
- [ ] S3: DAG 图以 SVG 渲染，并行步骤横向排列
- [ ] S4: 日志搜索高亮、自动滚动暂停/恢复、500 条截断均正常
