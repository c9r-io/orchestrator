# QA-120: GUI 连接韧性与系统通知

**关联设计**: DD-078
**关联 FR**: FR-068

## 场景 1: 连接向导 UI

**前置条件**: daemon 未运行

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 启动 GUI | 显示连接向导页面（非一行文字） |
| 2 | 检查向导内容 | 显示三项诊断：守护进程未启动 / 连接地址 / 证书问题 |
| 3 | 点击"重试连接" | 按钮变为"连接中..."，失败后显示错误信息 |
| 4 | 展开"手动配置" | 显示配置文件路径输入框和连接按钮 |
| 5 | 启动 daemon 后点击"重试连接" | 连接成功，进入主界面 |

## 场景 2: 自动重连

**前置条件**: GUI 已连接到 daemon

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 停止 daemon | 5 秒内顶部出现黄色横幅"连接中断，正在重连..." |
| 2 | 等待重连过程 | 横幅显示"尝试 1/3"、"尝试 2/3"、"尝试 3/3" |
| 3 | 在重连期间重启 daemon | 横幅变绿"已恢复连接"，3 秒后自动消失 |
| 4 | 确认数据恢复 | 角色信息重新探测，当前页面数据可用 |

## 场景 3: 重连失败

**前置条件**: GUI 已连接到 daemon

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 停止 daemon 并保持关闭 | 3 次重连后横幅变红"连接失败" |
| 2 | 横幅显示"重试"按钮 | 点击重试触发新一轮连接尝试 |

## 场景 4: 系统通知 — 任务完成

**前置条件**: GUI 已连接，已授权通知权限

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 通过 GUI 创建并启动任务 | 任务开始执行 |
| 2 | 开始 TaskWatch 监听 | 状态流正常接收 |
| 3 | 任务执行完成 | 收到 OS 通知"任务完成：「{name}」已成功完成" |
| 4 | GUI 在后台时重复步骤 3 | 仍然收到 OS 通知 |

## 场景 5: 系统通知 — 任务失败

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 创建一个会失败的任务 | 任务开始执行 |
| 2 | 任务失败 | 收到 OS 通知"任务失败：「{name}」执行失败" |

## 场景 6: 系统通知 — FR 草稿就绪

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在许愿池提交一个需求 | 创建 wish-pool 项目任务 |
| 2 | FR 草稿生成完成 | 收到 OS 通知"FR 草稿就绪：「{name}」的需求方案已生成，等待确认" |

## 场景 7: 错误信息人性化

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在 daemon 停止时执行任何 RPC | 显示"无法连接到服务器，请检查 daemon 是否运行" |
| 2 | 以 read_only 角色执行管理操作 | 显示"权限不足，需要更高级别的访问权限" |
| 3 | 查询不存在的资源 | 显示"未找到请求的资源" |
| 4 | 提交无效参数 | 显示"输入内容不符合要求: {detail}" |

## 场景 8: 流式连接韧性

**前置条件**: 正在 TaskWatch 流式监听

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 停止 daemon | 流断开，前端收到人性化错误消息 |
| 2 | 心跳检测到断连 | 触发自动重连流程 |
| 3 | 重连成功 | 用户可重新启动流式监听 |
