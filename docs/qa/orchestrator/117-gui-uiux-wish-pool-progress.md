# QA-117: GUI 用户界面 — 许愿池 + 进度观察 + 专家模式

**关联设计**: `docs/design_doc/orchestrator/74-gui-uiux-wish-pool-progress.md`
**关联 FR**: FR-064（已闭环）

## 验证场景

### 场景 1: 许愿池 — 输入与提交

**目标**: 非技术用户能用自然语言输入需求并提交。

**步骤**:
1. 打开 GUI，切换到「许愿池」Tab
2. 在文本区输入需求描述（如"我想要一个用户登录功能"）
3. 确认字符计数显示正确（最大 2000 字）
4. 点击「许愿」按钮提交
5. 使用 Cmd/Ctrl+Enter 快捷键提交

**期望**: 调用 `task_create(goal=输入)` 成功，自动跳转到许愿详情页。

---

### 场景 2: 许愿详情 — FR 草稿预览与确认

**目标**: 用户能查看 FR 草稿并确认开发。

**步骤**:
1. 进入许愿详情页
2. 查看原始需求文本
3. 等待 FR 草稿通过 TaskFollow streaming 实时生成
4. 点击「确认开发」→ 调用 TaskCreate 创建开发任务
5. 确认自动跳转到进度观察 Tab 并选中新任务

**期望**: 完成"许愿 → 审阅 → 确认"全流程 < 5 分钟。

---

### 场景 3: 进度观察 — 任务列表与实时更新

**目标**: 进度页面实时反映任务状态。

**步骤**:
1. 切换到「进度观察」Tab
2. 确认任务卡片显示：状态图标 + 任务名 + 进度条 + 时间
3. 确认排序：运行中 > 暂停 > 失败 > 已完成
4. 确认进度条 `finished_items/total_items` 正确

**期望**: 任务列表清晰展示所有任务的当前状态。

---

### 场景 4: 任务详情 — 功能按钮

**目标**: 操作按钮调用正确的 RPC 且受 RBAC 控制。

**步骤**:
1. 进入运行中任务 → 点击「暂停」→ 确认调用 `TaskPause`
2. 进入已暂停任务 → 点击「恢复」→ 确认调用 `TaskResume`
3. 进入失败任务 → 点击「重试」→ 确认调用 `TaskRetry`
4. admin 角色 → 点击「删除」→ 确认弹出确认对话框 → 确认调用 `TaskDelete`
5. read_only 角色 → 确认操作按钮不可见

**期望**: 按钮功能正确，RBAC 权限映射正确。

---

### 场景 5: 实时日志流

**目标**: TaskFollow streaming 在 GUI 中实时显示日志。

**步骤**:
1. 进入任务详情页
2. 点击「追踪」开始日志流
3. 确认日志带时间戳前缀实时追加
4. 确认自动滚动到底部
5. 点击「停止」终止流

**期望**: 日志延迟 < 2 秒。

---

### 场景 6: 专家模式 — 工作流 Tab

**目标**: 专家模式展示步骤 DAG 和状态。

**步骤**:
1. 任务详情页点击「专家」按钮
2. 确认按钮变为「专家 ✓」
3. 确认工作流 Tab 显示步骤列表：序号 + 状态图标 + 名称
4. 使用 Cmd+E 快捷键切换专家模式

**期望**: 工作流步骤与 TaskInfo items 一致。

---

### 场景 7: 专家模式 — 资源 Tab

**目标**: 资源浏览与编辑功能正确。

**步骤**:
1. 切换到「资源」Tab
2. 选择 kind 过滤按钮（workspaces/workflows/agents 等）
3. 确认显示资源 YAML 内容（ResourceGet）
4. operator 角色：点击「编辑」→ 修改 YAML → 点击「应用」（ResourceApply）

**期望**: 资源正确加载，编辑功能受 RBAC 控制。

---

### 场景 8: 专家模式 — Agent Tab

**目标**: Agent 管理功能正确。

**步骤**:
1. 切换到「Agent」Tab
2. 确认显示 Agent 表格：名称、状态、健康度、在途任务
3. admin 角色：点击 Cordon/Uncordon/Drain 按钮
4. 确认 Drain 操作弹出确认对话框
5. 非 admin 角色：操作按钮不可见

**期望**: Agent 管理受 RBAC 控制。

---

### 场景 9: 专家模式 — 原始数据 Tab

**目标**: 完整 TaskInfo JSON 可查看和复制。

**步骤**:
1. 切换到「原始数据」Tab
2. 确认显示完整 TaskInfo JSON（格式化缩进）
3. 点击「复制」按钮，确认 JSON 复制到剪贴板

**期望**: JSON 完整且可复制。

---

### 场景 10: 可访问性

**目标**: 通过 WCAG AA 可访问性要求。

**步骤**:
1. 确认所有按钮有 `aria-label`
2. 确认进度条有 `role="progressbar"` + `aria-valuenow`/`aria-valuemax`
3. 确认日志区有 `role="log"` + `aria-live="polite"`
4. 确认对话框有 `role="dialog"` + `aria-modal="true"`
5. 确认 Tab 键可遍历所有交互元素
6. 确认 `prefers-reduced-motion` 下动画禁用

**期望**: 所有 ARIA 属性正确，键盘导航完整。

---

### 场景 11: 键盘快捷键

**步骤**:
1. Cmd/Ctrl+1 → 切换到许愿池
2. Cmd/Ctrl+2 → 切换到进度观察
3. Cmd/Ctrl+Enter → 提交许愿
4. Escape → 返回上一层
5. Cmd/Ctrl+E → 切换专家模式（任务详情页）

**期望**: 所有快捷键响应正确。
